## What is Spectra?

Spectra is a **senior QE engineer's AI co-pilot**. Give it an OpenAPI spec; it returns a complete, risk-prioritised pytest test suite — no templates, no boilerplate, just context-aware tests generated by an LLM that understands security and boundary conditions.

```
OpenAPI Spec ──→ Parse ──→ Risk Score ──→ Generate Tests ──→ Deduplicate ──→ Report
     │              │           │                │                 │             │
 YAML/JSON     Structured    LLM tiers       LangChain        Embeddings    JSON/MD
               endpoints   (Critical→Low)   + Pydantic        cosine sim
```

## Features

| Feature | Details |
|---------|---------|
| **OpenAPI 3.x Parser** | Endpoints, path/query/body params, auth schemes, response schemas |
| **LLM Risk Scoring** | `Low / Med / High / Critical` per endpoint using LangChain + Pydantic structured output |
| **AI Test Generation** | `happy_path · boundary · auth_bypass · malformed_input` test types per risk tier |
| **Embedding Dedup** | Cosine similarity deduplication at configurable threshold (default `0.92`) |
| **Real-time UI** | React 19 SPA with live SSE pipeline streaming and run history |
| **LangSmith Tracing** | Full LLM call observability — prompts, latency, token cost |
| **CLI** | `qe-platform parse / risk-score / generate` for scripted pipelines |
| **Docker ready** | Multi-stage build: React → Python → test image |
| **CI/CD** | GitHub Actions: lint → unit tests → Docker build |

## Quick Start

### Prerequisites
- Python 3.10+
- Node.js 18+
- An Anthropic **or** OpenAI API key

### 1 — Clone & install

```bash
git clone https://github.com/pragatig25/spectra-qe.git
cd spectra-qe
python -m venv .venv && source .venv/bin/activate
pip install -e ".[all]"
```

### 2 — Configure

```bash
cp .env.example .env
```

Open `.env` and set **only the keys you have**:

```ini
# Required — choose one provider
ANTHROPIC_API_KEY=your-key-here      # recommended: claude-haiku-4-5 is fast & cheap
# or
OPENAI_API_KEY=your-key-here

LLM_PROVIDER=anthropic               # or openai
LLM_MODEL=claude-haiku-4-5

# Optional — for embeddings-based deduplication
EMBEDDING_MODEL=text-embedding-3-small

# Optional — for LangSmith tracing
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=your-key-here
```

> **Never commit `.env`** — it is in `.gitignore`.

### 3 — Start the platform

```bash
# Terminal 1 — backend
uvicorn qe_platform.api.app:app --reload --reload-dir qe_platform --port 8000

# Terminal 2 — frontend dev server (optional, for live HMR)
cd ui && npm install && npm run dev
```

Open **http://localhost:8000** → full-stack UI served by FastAPI.
Open **http://localhost:5173** → Vite dev server with HMR (proxies API to :8000).

### 4 — Docker (one command)

```bash
docker compose up app
# → http://localhost:8000
```

## Usage

### Web UI

Navigate to `http://localhost:8000` and use the sidebar:

| Page | What to do |
|------|-----------|
| **Dashboard** | Check API health, see run history |
| **Spec Parser** | Select a demo spec → Parse → inspect structured endpoints |
| **Risk Scorer** | Run LLM risk assessment → view tier distribution |
| **Test Generator** | Full pipeline → generated + deduplicated pytest cases |
| **Reports** | View exported test suites |

### CLI

```bash
# Parse a spec
qe-platform parse demo/petstore_openapi.yaml

# Score endpoint risk
qe-platform risk-score demo/petstore_openapi.yaml

# Generate full test suite (skip live execution)
qe-platform generate demo/petstore_openapi.yaml --skip-execution

# Run the built-in demo
qe-platform demo
```

### REST API

```bash
# Health check
curl http://localhost:8000/api/health

# List demo specs
curl http://localhost:8000/api/specs

# Parse a spec
curl -s -X POST http://localhost:8000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"spec_id": "petstore"}'

# Stream risk scoring (Server-Sent Events)
curl -N -X POST http://localhost:8000/api/risk-score/stream \
  -H "Content-Type: application/json" \
  -d '{"spec_id": "petstore"}'

# Stream full test generation
curl -N -X POST http://localhost:8000/api/generate/stream \
  -H "Content-Type: application/json" \
  -d '{"spec_id": "petstore", "skip_execution": true}'
```
